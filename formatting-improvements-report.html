🛠️ C语言格式化器逻辑改进报告

📋 改进概述

针对您提供的C代码格式问题，我们对格式化器进行了以下重要改进：

  • 改进大括号缩进逻辑，正确处理嵌套结构
  • 优化 else 语句的缩进对齐
  • 增强操作符格式化（赋值、比较、逗号等）
  • 改进关键字后的空格处理
  • 加强错误处理和安全性检查

🎯 测试用例

以下是您提供的需要格式化的C代码：

原始代码（格式不规范）
#include <stdio.h> #include <stdlib.h> int main(){ int x=10,y=20; if (x>y){ printf("x is greater\\n"); }else{ printf("y is greater\\n"); } for (int i=0;i<5;i++){ printf("Count: %d\\n",i); } return 0; }

⚡ 核心改进点

🔴 修复前的问题

  • 缩进不一致
  • 大括号位置混乱
  • else语句缩进错误
  • 操作符周围缺少空格
  • for循环格式不规范

🟢 修复后的效果

  • 统一4空格缩进
  • 大括号正确对齐
  • else与if同级对齐
  • 操作符规范间距
  • 循环结构清晰

🔧 技术改进详情

1. 缩进逻辑优化

新的缩进算法
// 主循环中的缩进处理 let lineIndentLevel = indentLevel // 如果行以}开头，减少缩进 if (line.startsWith('}')) { lineIndentLevel = Math.max(0, indentLevel - 1) } // 如果行以else开头，与对应的if保持同级 else if (line.startsWith('else')) { lineIndentLevel = Math.max(0, indentLevel - 1) }

2. 操作符格式化

新增的操作符处理
private formatOperators(line: string): string { // 在赋值操作符周围添加空格 line = line.replace(/([a-zA-Z0-9_])\\s*=\\s*([a-zA-Z0-9_])/g, '$1 = $2') // 在比较操作符周围添加空格 line = line.replace(/([a-zA-Z0-9_])\\s*>\\s*([a-zA-Z0-9_])/g, '$1 > $2') // 在逗号后添加空格 line = line.replace(/,\\s*/g, ', ') return line }

3. 大括号处理增强

改进的大括号逻辑
// 计算大括号对下一行缩进的影响 const openBraces = (line.match(/\\{/g) || []).length const closeBraces = (line.match(/\\}/g) || []).length // 更新下一行的缩进级别 newIndentLevel = currentIndent + openBraces - closeBraces
